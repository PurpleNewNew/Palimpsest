# AI 科研助手 - 页面设计

## 页面清单

| 页面       | 路径                             | 描述                           |
| ---------- | -------------------------------- | ------------------------------ |
| 课题列表   | `/`                              | 显示所有科研课题               |
| 课题概览   | `/project/:id`                   | 课题总览，信息面板             |
| 课题设置   | `/project/:id/settings`          | 课题配置（背景/目标等）        |
| AI 对话    | `/project/:id/chat`              | 课题级 AI 对话（复用OpenCode） |
| 代码项目   | `/project/:id/code`              | 代码项目管理                   |
| 代码详情   | `/project/:id/code/:codeId`      | 代码项目详情                   |
| 文献库     | `/project/:id/papers`            | 文献管理列表                   |
| 文献详情   | `/project/:id/paper/:paperId`    | 文献详情和原子提取             |
| 原子编辑器 | `/project/:id/atom/:atomId`      | 创建/编辑原子                  |
| 知识图谱   | `/project/:id/graph`             | 交互式图谱可视化               |
| 实验详情   | `/project/:id/experiment/:expId` | 实验执行详情/日志              |
| 实验历史   | `/project/:id/experiments`       | 实验记录列表                   |
| 服务器管理 | `/project/:id/servers`           | 远程服务器配置                 |
| 验证面板   | `/project/:id/validate`          | 批量验证管理                   |
| 设置       | `/settings`                      | 系统设置                       |

```
│  │     2. 稀疏注意力模式 - 减少计算量                 │   │
│  │     3. 滑动窗口注意力 - 平衡效率和效果             │   │
│  │                                                     │   │
│  │     您想深入了解哪个方向？我可以帮您生成假设...    │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  > 分析这个假设的可行性...                     [发送] │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**AI 对话特性**：

- **自动上下文注入**：对话开始时自动携带课题背景、目标、已有原子摘要
- **建议功能**：AI 主动建议可能的假设、研究方向
- **一键生成原子**：对话中可直接生成原子，存入知识网络
- **实验规划**：可讨论并规划验证实验

**复用 OpenCode**：

- 直接使用 OpenCode 原有对话界面
- 定制 System Prompt 包含课题信息
- 复用所有 Agent 和 Tool 能力

## 状态管理

基于 OpenCode 的 SolidJS store 模式：

```typescript
// 课题状态
interface ResearchState {
  projects: Project[]
  currentProject: Project | null
  atoms: Atom[]
  relations: Relation[]
  papers: Paper[]
  codeProjects: CodeProject[]
  experiments: Experiment[]
  servers: Server[]
}

// 全局状态
const [researchState, setResearchState] = createStore<ResearchState>({
  projects: [],
  currentProject: null,
  atoms: [],
  relations: [],
  papers: [],
  codeProjects: [],
  experiments: [],
  servers: [],
})
```
